B - 順列/Permutations

Time limit時間制限 : 2sec / Stack limitスタック制限 : 256MB / Memory limitメモリ制限 : 256MB

問題文

1 から N までの整数を並べ替えてできる順列 p1,…,pN が与えられます。 高橋君は、紙に N 個の点を書き、各 i(1≤i≤N) について i 番の点から pi 番の点への矢印を書きました。 このとき、矢印の輪がいくつかできます。 たとえば、順列が {2, 4, 5, 1, 3} のとき、1→2→4→1 の輪と 3→5→3 の輪の二つができます。 また、順列が {1, 2, 3, 4, 5} のときは、各点から自分自身への輪ができるので、矢印の輪が合計 5 個できます。 順列が与えられたとき、矢印の輪の個数を求めてください。

制約

  • 1≤N≤105
  • p1,…,pN は　1,…,N の並べ替えである

入力

入力は以下の形式で標準入力から与えられる。

N
p1 … pN

出力

矢印の輪の個数を出力せよ。

入力例1

5
2 4 5 1 3

出力例1

2

入力例2

5
1 2 3 4 5

出力例2

5

Question

You are given a permutation p1,…,pN that is a sequence of n distinct positive integers, each of them is from 1 to N. Mr. Takahashi writes N dots on a piece of paper and writes arrows from point i to point pi for each i(1≤i≤N). This gives a number of cycles with arrows. For example, for the permutation {2, 4, 5, 1, 3}, you will get the two cycles 1→2→4→1 and 3→5→3. Furthermore, for the permutation {1, 2, 3, 4, 5}, you will get a cycle from each point back to that point, for a total of five cycles. Determine the number of cycles for the permutation you are given.

Constrains

  • 1≤N≤105
  • p1,…,pN is a permutation of 1,…,N

Input

Inputs are provided from standard inputs in the following form.

N
p1 … pN

Output

Output the number of cycles of arrows.

Input Example #1

5
2 4 5 1 3

Output Example #1

2

Input Example #2

5
1 2 3 4 5

Output Example #2

5
